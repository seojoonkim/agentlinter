# AgentLinter v2.4.0 — 기획 초안 (2026-04-19)

> 주제: **"정확도 × 확장성"** — v2.3.0에서 완성된 60+ rule 기반 위에, 에이전트 설계의 *진짜* 문제를 잡는 심층 린팅.

---

## 1. 목표

v2.3.0까지는 **정적 분석의 완성도**를 끌어올렸음 (60+ rules, 11 카테고리, 다중 프레임워크).  
v2.4.0은 다음 세 가지를 한 단계 더 파내려감:

1. **에이전트 보안 심화** — AI agent 특화 공격 벡터 (HiveFence 통합)
2. **멀티 에이전트 구조 린팅** — orchestration, sub-agent spawn, streamTo 규칙
3. **런타임 신호 통합** — 실행 로그/메트릭을 정적 분석에 feed (선택적)

**v2.4.0은 "rules 수 증가"가 아닌 "rules 깊이 증가"가 핵심.**

---

## 2. 제안 기능 (우선순위 순)

### 🔴 #1. Agent Security+ (HiveFence 병합)

**현재:** security 카테고리 25 pattern
**v2.4.0:** 50+ pattern, HiveFence 방어 패턴 정식 병합

**세부:**
- Prompt injection hardening: 15 → 30 pattern (jailbreak, supply-chain injection, memory poisoning, action gate bypass, unicode steganography)
- Credential leak: AWS/JWT/API key 외 **Cloud provider tokens** (Vercel, Supabase, GoDaddy 등) 확장
- Cross-agent attack: 5남매 구조에서 악성 sub-agent spawn 탐지 (`sessions_spawn` 파라미터 검증)
- PII DLP: 10개 → 20개 패턴 (한국어 주민번호, 사업자번호 등 로컬라이즈)

**새 규칙 후보:**
- `security/cross-agent-privilege-escalation` — sub-agent가 parent 권한 초과 요청
- `security/unicode-steganography` — zero-width 문자 / 우회 인코딩
- `security/action-gate-bypass` — 외부 액션(메일, 트윗)이 확인 절차 없이 호출

**레퍼런스:** `~/сlawd/skills/prompt-guard/` (650+ 패턴), `HiveFence v1.0.0`

### 🔴 #2. Multi-Agent Orchestration Linter

**배경:** 5남매(Zeon/Sion/Mion/Sano/Raon) 같은 멀티 에이전트 구조가 2026년 이후 사실상 표준. 하지만 현재 AgentLinter는 **단일 에이전트** 가정으로 설계됨.

**새 카테고리:** `orchestration`

**세부 규칙:**
- `orchestration/subagent-spawn-pattern` — `sessions_spawn` 호출에 `streamTo="parent"` 누락 감지 (AGENTS.md TIER 1 규칙 기반)
- `orchestration/model-routing-defined` — self-calibration 라우팅 (haiku/sonnet/opus) 정의 여부
- `orchestration/shared-context-paths` — 공유 컨텍스트 파일 경로 검증 (`shared-context/active-tasks.md` 등)
- `orchestration/completion-signal` — 무거운 작업에 완료 신호 파일 패턴 사용 여부 (`/tmp/xxx-done.txt`)
- `orchestration/silent-mode-policy` — 그룹챗 응답 규칙 정의 여부

**지원 아키텍처:**
- Claude Code multi-agent (5남매, seojoonkim 스타일)
- OpenAI Assistants API + Agents SDK
- LangGraph, CrewAI 등 (detection heuristics)

### 🟡 #3. Runtime Signal Integration (Opt-in)

**컨셉:** 정적 린팅 결과를 **실제 에이전트 실행 로그**와 대조하여 "쓰이지 않는 규칙" / "자주 위반되는 규칙" 자동 감지.

**세부:**
- `agentlinter profile --log <path>` — 실행 로그 파싱 (Claude Code session log, OpenAI Assistants run log)
- 규칙별 "활성도" 점수: 지난 30일 동안 실제 어긴 횟수
- **Dead rule detection** — 정의돼 있지만 실제 실행에서 한 번도 참조되지 않은 규칙 → 제거 권고
- **Rule hotness** — 자주 위반되는 규칙 → 더 강하게 강조 필요

**데이터 소스 (예시):**
- `~/сlawd/memory/YYYY-MM-DD.md` 패턴 (daily log)
- `compound/lessons.md` (교정 기록)

**이건 opt-in** — 기본 동작은 영향 없음, `--profile` 플래그 명시 시만.

### 🟢 #4. Git Pre-commit Integration

**명령:** `npx agentlinter install-hook`

**동작:**
- `.husky/pre-commit`에 `agentlinter --since HEAD~1` 자동 추가
- CLAUDE.md / AGENTS.md 수정 시 자동 린팅
- 점수 하락 시 커밋 블록 (옵션)

**부가:** `--since` 플래그 — git diff 기반 변경 파일만 린팅 (속도 개선)

### 🟢 #5. Report Page v2 — Diff Mode

**컨셉:** 이전 리포트 대비 점수 변화 시각화

**UI:**
- 점수 추이 (sparkline)
- 새로 발생한 diagnostics vs 해결된 diagnostics
- "Regression alert" — 점수가 5점 이상 하락 시

**구현:** 기존 탭 구조에 "Timeline" 탭 추가
