## 3. Detailed Implementation Specs

### 3.1 Token Bloat Score 강화 (P0)

**File:** `packages/cli/src/engine/rules/tokenEfficiency.ts`
**Action:** Enhance existing file (add 3 new sub-rules)

#### New Sub-rules:

**a) `clarity/duplicate-content` — 중복 내용 탐지**
- 로직: 각 파일의 섹션(heading 단위)을 비교, 유사도 높은 섹션 쌍 탐지
  - 같은 파일 내 섹션 간 비교 + 파일 간 비교
  - 3-gram 기반 Jaccard similarity ≥ 0.6이면 경고
- Severity: `warning`
- Fix: "섹션 A와 B가 60%+ 유사합니다. 하나로 통합하세요."

**b) `clarity/obvious-statements` — "당연한 말" 감지**
- 로직: 패턴 매칭으로 LLM에게 불필요한 지시 탐지
  ```
  /be accurate/i, /respond correctly/i, /follow instructions/i,
  /be professional/i, /do a good job/i, /be thorough/i,
  /make sure to/i (+ 뒤에 obvious action), /remember to/i,
  /don't forget to/i, /please note that/i,
  /정확하게 답변/i, /지시를 따르/i, /실수하지 마/i
  ```
- Severity: `info`
- Fix: "이 지시는 LLM의 기본 행동입니다. 제거해도 성능에 영향 없습니다."

**c) `clarity/token-budget-range` — 토큰 수 범위 체크 강화**
- 현재: line count만 체크 (A/B/C/D)
- 변경: 실제 토큰 수 추정 (1 token ≈ 4 chars for English, ≈ 1.5 chars for Korean)
  - A: ≤ 2,000 tokens
  - B: ≤ 5,000 tokens
  - C: ≤ 10,000 tokens
  - D: > 10,000 tokens
- 기존 line-based 그레이딩도 유지 (backward compat)
- 신규 diagnostic에 추정 토큰 수 표시

### 3.2 Freshness / Staleness Detector (P0)

**File:** `packages/cli/src/engine/rules/freshness.ts` (NEW)
**Registry:** `packages/cli/src/engine/rules/index.ts`에 추가

#### Sub-rules:

**a) `consistency/stale-file-reference` — 파일 참조 검증**
- 로직:
  1. Markdown 내 경로 패턴 추출: `` `path/to/file` ``, `~/path`, `./path`, backtick 내 경로
  2. workspace 기준으로 `fs.existsSync()` 체크
  3. 없으면 warning
- Severity: `warning`
- Fix: "참조된 파일 `{path}`가 존재하지 않습니다. 경로를 업데이트하거나 참조를 제거하세요."

**b) `consistency/stale-date` — 오래된 날짜 탐지**
- 로직:
  1. `YYYY-MM-DD`, `last_updated:`, `updated:` 패턴 추출
  2. 현재 날짜와 비교, 90일+ 이전이면 warning, 180일+ 이면 error
- Severity: `warning` / `error`
- Fix: "이 날짜({date})는 {N}일 전입니다. 내용이 최신인지 확인하세요."

**c) `consistency/stale-package-reference` — 패키지 참조 검증**
- 로직:
  1. Markdown 내 패키지명 패턴 추출 (npm install X, package.json 참조)
  2. workspace의 `package.json`과 크로스체크
  3. dependencies에 없으면 info
- Severity: `info`
- Fix: "참조된 패키지 `{pkg}`가 package.json에 없습니다."

### 3.3 @ Import Validator (P1)

**File:** `packages/cli/src/engine/rules/importValidator.ts` (NEW)
**Registry:** `packages/cli/src/engine/rules/index.ts`에 추가

#### Sub-rules:

**a) `structure/dead-import` — @file.md 참조 검증**
- 로직:
  1. CLAUDE.md 등에서 `@path/to/file.md` 패턴 추출
  2. Claude Code import 문법: `@AGENTS.md`, `@compound/services.md`
  3. 실제 파일 존재 여부 체크
- Severity: `warning`
- Fix: "Import `@{path}`가 참조하는 파일이 존재하지 않습니다."

**b) `structure/circular-import` — 순환 참조 탐지**
- 로직:
  1. 모든 파일의 @import를 수집해 directed graph 구성
  2. DFS로 cycle 탐지
  3. cycle 발견 시 error
- Severity: `error`
- Fix: "순환 참조 발견: {A} → {B} → {A}. import 구조를 정리하세요."

### 3.5 Describe vs Command 분류기 (P2 — Partial)

**File:** `packages/cli/src/engine/rules/skillsVsCommands.ts` (MODIFY)
**Action:** LLM 의존 없이 패턴 기반으로 구현 (v2.2에서 LLM 버전 추가 가능)

#### Enhancement:

**a) `claude-code/descriptive-ratio` — 서술 vs 명령 비율**
- 로직:
  1. 각 문장 추출 (마침표/줄바꿈 기준)
  2. Imperative 패턴: 동사로 시작 (`Use`, `Run`, `Always`, `Never`, `Do`, `Don't`, ...)
  3. Descriptive 패턴: `is`, `are`, `was`, `has`, `The`, `This`, `It`, 주어+동사 구조
  4. Descriptive 비율 > 60%이면 warning
- Severity: `warning`
- Fix: "서술문이 {N}%입니다. Agent 지시 파일은 명령형(imperative) 문장이 효과적입니다. 'The system does X' → 'Do X' 형식으로 변경하세요."
